# 開発部 デブログ (2026-03-21)

## 作業内容
- リサーチ部の「Google Calendar CLI Integration 技術調査報告」に基づき、カレンダー連携のプロトタイプを実装。
- 機密情報（`credentials.json`, `token.json`）を安全に管理するためのディレクトリ `.company/secret/` を作成。
- Git 管理から除外するため、`.gitignore` に `.company/secret/` を追加。
- Google Calendar API を使用する Python スクリプト `.company/bin/sync_calendar.py` を作成。
- 秘書室の依頼に基づき、LINE ボット（`linebot_gas`）を拡張し、「日記取り込み機能」を実装。

## 実装詳細 (LINE Bot 日記取り込み)
- `/id` コマンドを追加：現在のチャット（グループ/トーク）の ID を返却する機能。
- `/set_diary_group` コマンドを追加：実行したグループを「日記用グループ」として `PropertiesService` に登録する機能。
- `saveMediaToDriveAsync` の拡張：日記用グループからの画像メッセージを、通常フォルダではなく Google ドライブ上の `Gemini_Journal_Scans` フォルダに保存する分岐ロジックを実装。
- `getDiaryFolder()` ヘルパー関数の追加：`Gemini_Journal_Scans` フォルダを自動検索・作成し、フォルダ ID を永続化。
- `clasp push` によりデプロイ完了。

## 実装詳細 (`sync_calendar.py`)
- `google-api-python-client`, `google-auth-oauthlib` を使用。
- 直近 7 日間の予定を `primary` カレンダーから取得。
- 出力形式は構造化 JSON。
- 依存ライブラリ未インストールの際の案内機能。
- 初回認証時のブラウザ起動通知。

## 実装詳細 (`sync_journal_drive.py`)
- `google-api-python-client` を使用し、Google ドライブから日記画像を同期するスクリプトを作成。
- `Gemini_Journal_Scans` フォルダ内の画像（png, jpg, jpeg）を `.company/secretary/inbox/scans/` に一括ダウンロード。
- ダウンロード成功後、二重取り込み防止のため、ドライブ上のファイルを `Gemini_Journal_Archive` フォルダへ移動。
- 既存の `credentials.json` と `token.json` を再利用。
- 必要に応じて `https://www.googleapis.com/auth/drive` スコープを追加（初回実行時に再認証が必要）。
